# Docker 开发环境服务说明

所有服务默认绑定在 `127.0.0.1`，只给本机访问。Compose 里已设置 `restart: unless-stopped`，容器创建成功后会随 Docker daemon 自动恢复。

## 一键命令

```bash
cd /home/xs/workplace/docker
./scripts/pull-all-images-cn.sh
./scripts/up-all.sh
./scripts/ps-all.sh
```

`up-all.sh` 会一次性启动全部 20 组 Compose 服务，并自动处理这些需要额外初始化的开发集群：

- `nacos`：导入 Nacos MySQL schema，然后重启 Nacos。
- `redis-cluster-6`：在 6 个 Redis 节点之间创建 3 主 3 从分片集群。

## 状态检查

```bash
cd /home/xs/workplace/docker
docker compose ls --all
./scripts/ps-all.sh
```

关键连通性检查：

```bash
curl http://127.0.0.1:18080/
curl http://127.0.0.1:18848/nacos/
curl http://127.0.0.1:19090/-/healthy
curl http://127.0.0.1:19000/minio/health/live
docker exec dev-redis-cluster-1 redis-cli -a xsailxma cluster info
docker exec dev-mongo-replica-1 mongosh --quiet --eval 'rs.status().members.map(m => m.name + ":" + m.stateStr).join("\n")'
```

## 基础单机服务

| 目录 | 作用 | 常用入口 |
| --- | --- | --- |
| `redis-local` | 单机 Redis，适合普通缓存、队列、本地开发。 | `127.0.0.1:16379` |
| `mysql-local` | 单机 MySQL 8，适合普通业务库开发。 | `127.0.0.1:13306` |
| `postgres-local` | 单机 PostgreSQL 16，适合 PG 项目开发。 | `127.0.0.1:15432` |
| `mongo-local` | 单机 MongoDB 7，适合普通文档库开发。 | `127.0.0.1:37017` |
| `rabbitmq-local` | 单机 RabbitMQ，带管理后台。 | AMQP `127.0.0.1:15672`，管理端 `http://127.0.0.1:15673` |
| `nginx` | 独立 Nginx 容器，可做本地反代或静态站点。 | `http://127.0.0.1:18080`，`https://127.0.0.1:18443` |

## 消息、流式处理

| 目录 | 作用 | 常用入口 |
| --- | --- | --- |
| `kafka-local` | 单节点 Kafka，带 Kafka UI，日常开发优先用这个。 | Kafka `127.0.0.1:19092`，UI `http://127.0.0.1:18082` |
| `kafka-cluster` | 三节点 Kafka KRaft 集群，固定同一个 `KAFKA_KRAFT_CLUSTER_ID`，适合测试分区、副本、集群行为。 | 节点端口 `19093/19094/19095`，UI `http://127.0.0.1:18081` |
| `rabbitmq-cluster` | 三节点 RabbitMQ 集群，适合集群队列和高可用测试。 | 节点端口 `25672/35672/45672`，管理端 `http://127.0.0.1:25673` |

## 集群和高可用数据库

| 目录 | 作用 | 常用入口 |
| --- | --- | --- |

| `redis-sentinel-6` | Redis 主从 + 3 Sentinel，适合哨兵切换测试。 | Redis `16410/16411/16412`，Sentinel `26410/26411/26412` |
